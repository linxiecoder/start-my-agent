# 什么是智能体
定义：通过**传感器**，可以感知**环境**，**自主**通过**执行器**采取**行动**以达成特定目的的实体

## 传统意义下的智能体
+ 反射智能体：由规则“条件-动作”组成决策核心。但是无法根据环境来识别条件
+ 基于模型的反射智能体：智能体内部有“世界模型”，可以基于更连续、完整的世界状态去识别条件
+ 基于目标的智能体：不再被动根据环境去识别条件，而是主动、有预见的选择导向某个特定未来状态的行动。核心能提体现在对未来的规划上
+ 基于效用的智能体：在相互冲突的目标之间权衡，不再简单地达成某个目标，而是最大化效用
+ 学习型智能体：通过强化学习，不断修正决策策略

## 大语言模型驱动的新范式
传统的智能体的能力来源于工程师显示编程和能力构建，行为模式确定且有边界

LLM智能体通过在海量数据上预训练，获得隐式的世界模型与强大的涌现能力

工作方式：规划与推理---工具使用---动态修正

新范式：模型训练、提示工程、微调

## 智能体的类型
+ 基于内部决策架构的分类
+ 基于时间与反应性的分类：追求速度的反应性和追求最优解的规划性之间的平衡
    - 反应式智能体：对环境刺激做出即时反应，速度快、计算开销低。包含简单反应式和基于模型的智能体
    - 规划式智能体：在行动前会思考和规划。不会对感知做出即时反应，需要利用内部世界模型，探索和评估未来各种可能性，寻求最优解。包含基于目标和基于效用的智能体
    - 混合式智能体：既需要即时反应，有需要长期规划。经典的混合架构设计是：底层是快速反应的模块，处理紧急情况和基本动作；高层是规划模块，负责制定长远的规划。现代的智能体则是将两种模式混合，通常在“思考-行动-观察”中循环
        * 规划：在思考阶段，LLM根据环境做出下一步合理行动
        * 反应：在行动和观察阶段，智能体与外部工具和环境交互，并立即获得反馈
+ 基于知识表示的分类
    - 符号主义AI
    - 亚符号主义AI
    - 神经符号主义AI



# 智能体的构成和运行原理
## 任务环境定义
人工智能领域，使用PEAS模型描述任务环境。分析其性能度量（Performance）、环境（Environment）、执行器（Actuators）和传感器（Sensors）

环境是部分可观察的。环境可分为确定性和随机性。所有任务都发生在序贯且连续的环境中。

## 智能体的运行机制
智能体并非一次性完成任务，而是通过循环和环境交互，称为智能体循环（Agent Loop）

+ 感知：循环起点。智能体通过其传感器（比如用户输入接口）接收环境的输入信息，即观察。
+ 思考：接收到感知的信息后，智能体进入决策阶段。LLM中，这部分是大模型内部推理过程。主要分为两个阶段
    - 规划：智能体基于当前观察和内部记忆，更新对任务和环境的理解，并制定或者调整一个行动计划。可能会涉及到复杂任务拆分为子任务
    - 工具选择：根据当前计划，从可用工具库选择最适合进行下一步的工具
+ 行动：决策完成后，智能体通过其执行器（Acutuators）执行具体行动。

行动并非循环终点。智能体的行动会引发环境的状态变化，环境随机会产生一个新的感知作为结果的反馈。



## 智能体的感知与行动
通过交互协议来规范智能体和环境的交互。

即，智能体每次输出的定义都是结构化的，遵循特定格式的文本。包含两个核心部分：

:::info
Thought（思考）：智能体内部决策的“快照”，以自然语言描述了智能体如何分析当前环境、回顾上一步观察结果、进行自我反思和问题分解、并最终规划下一步的具体行动

Action（行动）：智能体基于思考后，决定对环境施加的操作，一般是函数调用

:::

<font style="color:rgb(31, 35, 40);">例如，一个正在规划旅行的智能体可能会生成如下格式化的输出：</font>

```plain
Thought: 用户想知道北京的天气。我需要调用天气查询工具。
Action: get_weather("北京")
```

<font style="color:rgb(31, 35, 40);">这里的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Action</font>`<font style="color:rgb(31, 35, 40);">字段构成了对外部世界的指令。一个外部的</font>**<font style="color:rgb(31, 35, 40);">解析器 (Parser)</font>**<font style="color:rgb(31, 35, 40);"> </font><font style="color:rgb(31, 35, 40);">会捕捉到这个指令，并调用相应的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">get_weather</font>`<font style="color:rgb(31, 35, 40);">函数。</font>

<font style="color:rgb(31, 35, 40);">行动执行后，环境会返回一个结果。例如，</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">get_weather</font>`<font style="color:rgb(31, 35, 40);">函数可能返回一个包含详细天气数据的 JSON 对象。然而，原始的机器可读数据（如 JSON）通常包含 LLM 无需关注的冗余信息，且格式不符合其自然语言处理的习惯。</font>

<font style="color:rgb(31, 35, 40);">因此，感知系统的一个重要职责就是扮演传感器的角色：将这个原始输出处理并封装成一段简洁、清晰的自然语言文本，即观察。</font>

<font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">Observation: 北京当前天气为晴，气温25摄氏度，微风。</font>

<font style="color:rgb(31, 35, 40);">这段</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Observation</font>`<font style="color:rgb(31, 35, 40);">文本会被反馈给智能体，作为下一轮循环的主要输入信息，供其进行新一轮的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Thought</font>`<font style="color:rgb(31, 35, 40);">和</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Action</font>`<font style="color:rgb(31, 35, 40);">。</font>

<font style="color:rgb(31, 35, 40);">综上所述，通过这个由 Thought、Action、Observation 构成的严谨循环，LLM 智能体得以将内部的语言推理能力，与外部环境的真实信息和工具操作能力有效地结合起来。</font>

# 智能体应用的协作模式
## 作为开发者工具的智能体
+ GitHubCopilot：深度集成VS，以代码补全闻名
+ Claude Code：AI编程助手，旨在通过自然语言协助开发者在完成编码任务。可以理解完整的代码库结构，执行代码编辑、调试、测试。
+ Trae：专注为开发者提供智能化的代码生成和优化服务。轻量级设计和快速响应，适合频繁迭代和原型开发
+ Cursor：与上述插件或集成功能而存在的工具不同，他是编辑器。具备顶级的代码生成和聊天能力，具备理解代码库上下文

## 作为自主协作者的智能体
像项目成员独立规划、推理、执行和反思，直到最后交付。

+ **单智能体自主循环**：AgentGPT。核心是一个智能体通过“思考-规划-执行-反思”的闭环，不断地自我提示和迭代
+ **多智能体协作**：主流的探索方向。
    - 角色扮演式对话：CAMEL框架，通过为两个智能体设置不同的角色和沟通协议，让他们在结构化对话中完成工作
    - 组织化工作流：MetaGPT和CrewAI。模拟一个“分工明确的团队”每个智能体都有预设的职责和工作流程（SOP），通过层级化或顺序化
+ **高级控制流架构**：诸如LangGraph框架，侧重为智能体提供强大的底层工程基础。将智能体执行过程建模为状态图，从而更灵活的实现循环、分支、回溯及人工介入等复杂场景。



## workflow和Agent的差异
workflow是让AI按部就班的执行命令，Agent是赋予AI自由度去完成目标
