# 3.1 语言模型与Transfermer架构
## 3.1.1 从N-gram到RNN
语言模型（Language Model，LM）是自然语言处理的核心，根本任务是计算一个词序列（一个句子）出现的概率。一个好的语言模型可以告知我们什么句子是通顺的自然地。在多智能体系统中，语言模型是智能体理解人类指令、生成回应的基础。

### （1）统计语言模型与N-gram思想
在深度学习兴起前，统计方法是语言模型的主流。核心思想是：一个句子出现的概率，等于该句子中每个词出现的条件概率的连乘。对于一个由词$ w_1,w_2...w_n $构成的句子S，其概率P(S)可以表示为：

$ P(S)=P(w_1,{W_2}...W_n)=P(w_1)*P(W_2|w_1)*P(w_3|{w_1,W_2})...P(W_n|w_1,W_2...W_{n-1}) $

这个公式被称为概率的链式法则。

这个公式基本不可计算，因为$ P(W_n|w_1,W_2...W_{n-1}) $这种条件概率太难出现，甚至在训练数据中都没有

因此引入**马尔科夫假设**。核心思想是：可以不必回溯一个词的全部历史，而是近似的认为一个词出现的概率只与他前面有限的m-1个词有关，基于这个假设建立的模型叫做N-gram模型。

Bigram（N=2）：$ P(W_n|w_1,W_2...W_{n-1}) $$ \approx $$ P(W_n|W_{n-1}) $

Trigram（N=3）：$ P(W_n|w_1,W_2...W_{n-1}) $$ \approx $$ P(W_n|W_{n-2},W_{n-1}) $

这个概率的计算成为**最大似然估计**（Maximum Likelihood Estimatio, MLE）

$ P(W_n|W_{n-1})=\frac{Count(W_{i-1},W_i)}{Count(W_{i-1}} $

其中，$ Count(W_{i-1},W_i)
 $表示词对$ (W_{i-1},W_i)
 $在语料库中连续出现的次数

$ Count(W_{i-1} $表示单个词$ W_{i-1}
 $在语料库中出现的总次数


假设我们拥有一个仅包含以下两句话的迷你语料库：datawhale agent learns,datawhale agent works我们的目标是：使用 Bigram (N=2) 模型，估算句子 datawhale agent learns出现的概率。根据 Bigram 的假设，我们每次会考察连续的两个词（即一个词对）。

答案是0.5

N-gram有两个致命缺陷：

+ 数据稀疏性（Sparsity）：如果一个词在语料库中从未出现，其概率估计为0，不合理。可以通过平滑（Smoothing）缓解，但是无法根除。
+ 泛化能力差：模型无法理解词与词之间的语义相似性。

### （2）神经网络语言模型与词嵌入
N-gram模型的根本缺陷在于将词视为孤立、离散的符号。

为了解决这个问题，研究者转向神经网络，提出一种思想：用连续的向量表示词。2003年提出**前馈神经网络模型**（Freedforward Neural Network Language Model，FNNLM）

核心思想分为2步：

+ **构建一个语义空间**：创建一个高维的连续向量空间，然后将词汇表中每个词都映射为该空间的一个点。这个点（即向量）被称为词嵌入（Word Embedding）或词向量。在这个语义空间中，语义上相近的词，对应的向量位置也很近。
+ **学习从上下文到下一个词的映射**：利用神经网络强大的拟合能力，来学习一个函数。这个函数的输入是前n-1个词的词向量，输出是词汇表中每个词在当前上下文后出现的概率。

![](https://cdn.nlark.com/yuque/0/2026/png/22681858/1774774776319-a4276989-a835-4bbf-b45a-20939d2a99a9.png)

如图，这个架构中，词嵌入是在模型训练过程中自动学习得到的。模型为了完成“预测下一个词”的任务，会不断调整每个词的向量位置，最终使这些向量具备丰富的语义信息。将词转换为向量后，可以用数学工具来度量他们的关系。最常用的是余弦相似度（Cosine Similary），通过计算两个向量的夹角的余弦值衡量他们的相似性。

$ similary(\vec {a},\vec {b})=cos(θ)=\frac{\vec{a}*\vec{b}}{|\vec{a}||\vec{b}|} $

这个公式的含义是：

+ 如果两个向量的方向相同，夹角为$ 0^° $，余弦值为1，表示完全相关
+ 如果两个向量的方向正交（垂直），夹角为$ 90^° $，余弦值为0，表示毫无关系
+ 如果两个向量的方向相反，夹角为$ 180^° $，余弦值为-1，表示完全负相关

案例：<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">vector('King') - vector('Man') + vector('Woman')</font>

前馈神经网络模型通过词嵌入，成功解决了N-gram模型泛化能力差的问题。但是仍然有缺陷，上下文窗口固定，只能考虑固定数量的前文，为能够处理任意长序列的循环神经网络埋下了伏笔。

### （3）循环神经网络（RNN）和长短时记忆网络（LSTM）
循环神经网络（Recurrent Netural Network,RNN），其核心思想：为网络增加“记忆”能力

RNN的设计引入了一个隐藏状态（hidden state）向量，可以理解为网络的短期记忆。在处理序列的每一步，网络都会读取当前输入词，并结合它上一步的记忆（即上一步的隐藏状态），然后生成一个新的记忆（即当前步的隐藏状态）传递给下一刻。

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/22681858/1774775996363-344d470a-2bae-469c-a587-30767abd8e7a.png)

标准的RNN实践过程中存在一个严重的问题：长期依赖问题（Long-term Dependency Problem）。在训练过程中，模型需要通过反向传播算法根据输出端的误差来调整网络深处的权重。对于RNN而言，序列的长度就是网络深度。当序列很长时，梯度在从后向前传播的过程中会经过多次连乘，导致梯度消失或梯度爆炸。

梯度消失导致模型无法有效学习到序列早期信息对后期输出的影响，即难以捕捉长距离的依赖关系。

为了解决长期依赖问题，长短时记忆网络（Long Short-Term Memory，LSTM）被设计出来。LSTM是一种特殊的RNN，其核心创新在于引入了细胞状态（Cell State）和门控机制（Gating Mechanism）。

细胞状态可以看做是隐藏状态的信息通路，允许信息在时间步之间更顺畅传递。  

门控机制有几个小的神经网络组成，可以学习怎么有选择的让信息通过，从而控制细胞状态中信息的增加和移除。

遗忘门（forget gate）：决定从上一时刻的细胞状态中丢弃哪些信息。

输入门（Input gate）：决定将当前输入中哪些新信息存入细胞状态。

输出们（Output gate）：决定根据当前的细胞状态，输出哪些信息到隐藏状态。

## 3.1.2 Transformer架构解析
RNN和LSTM引入循环结构处理序列数据，但是必须按顺序处理，无法进行大规模并行计算。

2017年，谷歌团队提出Transformer，完全抛弃了循环架构，改为完全依赖注意力（Attention）机制，实现了真正意义上的并行计算。

### （1）Encoder-Decoder整体结构
最初Transformer是为了端到端任务机器翻译设计，宏观上遵循了编码器-解码器（Encoder-Decoder）架构。

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/22681858/1774835927231-189d6c34-f8dd-41a3-bd84-ae92feaf7b65.png)

+ **编码器（Encoder）**：任务是“理解”输入的整个句子。会读取所有输入的词元，最终为每个词元生成一个富含上下文信息的向量表示。
+ **解码器（Decoder）**：任务是“生成”目标句子。会参考自己已经生成的千问，并“咨询”编码器理解的结果，来生成下一个词。

### （2）自注意力机制到多头注意力
**自注意力（Self-Attention）机制**，允许模型在处理序列中的每一个词时，都能兼顾句子中的所有其他词，并为这些词分配不同的“注意力权重”。权重越高的词，代表其与当前词的关联性越强，其信息也在当前词中占据更大的比重。

自注意力机制为每个输入的词元向量引入了三个可学习的角色：

+ 查询（Query，Q）：代表当前词元，正在主动地“查询”其他词元以获得信息。
+ 键（Key，K）：代表句子中可被查询的词元“标签”或“索引”
+ 值（Value，V）：代表词元本身所携带的“内容”或“信息”

这三个词都是由原始的词嵌入向量乘以不同的、可学习的权重矩阵（$ W^θ,W^K,W^V $）得到的。整个计算过程可以分为以下几步：

+ 对于句子中每个词，都通过权重矩阵生成其QKV向量（q_i = x_i * W ^ q）
+ 计算相关性得分：要计算词A的新表示，就用词A的Q向量，去和句子中所有词（包括A自己）的K向量进行点积运算。这个得分反映了其他词对于理解词A的重要性(q_i * k__i*q_i*k_{i+1}...=Q_i*K^T)
+ 稳定化与归一化：将得到的**所有分数(q_i*K^T+q_{i+1}*k^T...=QK^T)**除以一个缩放因子$ \sqrt{d_k} $，其中$ d_k $是K向量的维度，以防止梯度过小，然后用Softmax函数将分数转换为总和为1的权重，也就是归一化的过程。
+ 加权求和：将上一步的权重分别乘以每个词对应的V向量，然后将所有结果相加。最终得到的向量，就是词A融合了全局上下文信息后的新表示。

可以用如下公式概括：

$ Attention(Q,K,V)=Softmax(\frac{QK^T}{\sqrt{d_k}})V $

如果只进行一次上述的注意力计算（即单头），模型可能只会关注一种类型的关联。比如处理“it”时，只关注了主语。但实际可能还要关注指代、时态等。

**多头注意力机制**应运而生，核心思想是：把一次做完变成分成几组，分开做，再合并。

他将原始的QKV向量，在维度上切成h份（h为头数），每一份都独立地进行一次单头注意力计算。最后将多个输出向量拼接，在通过一个线性变换进行整合。

### <font style="color:rgb(15, 17, 21);">（3）前馈神经网络</font>
多头注意力子层之后跟着一个**逐位置前馈网络（Position-wise-Feed-Forward Network,FFN）**。逐位置前馈神经网络的作用是从注意力层动态地聚合的信息中提取更高阶的特征。

“逐位置”意味着这个前馈网络会独立地作用于序列中的每个词元向量。换句话讲，对于一个长度为seq_len的序列，这个FFN会被调用seq_len次，每次处理一个词元。所有位置共享的是同一组网络权重。

$ FFN(X)=max(0,xW_1+b_1)W_2+b_2 $

其中，x是注意力子层的输出。w_1/b_1/w_2/b_2是可学习的参数。通常，第一个线性层的输出维度d_ff会远远大于输入的维度d_model，经过ReLU激活后再通过第二个线性层映射回d_model维度。这种“先放大再缩小”的模式被认为有助于模型学习更丰富的特征表示。

### （4）残差连接与层归一化
在Transformer的每个编码器和解码器层中，所有的子模块都被一个Add & Norm操作包裹。这个组合是为了保证Transformer能够稳定训练。

+ **残差连接（Add）**：将子模块的输出x直接加到该子模块的输出sublayer(x)上。这一结构解决了深度神经网络**梯度消失**（Vanishing Gradients）问题。在反向传播时，梯度可以绕过子模块直接向前传播，从而保证了即使网络层数很深，模型也能得到有效的训练。$ \text{Output} = x + \text{Sublayer}(x) $
+ **层归一化（Norm）**：该操作对单个样本所有的特征进行归一化，使其均值为0，方差为1。为了解决模型训练过程中的**内部协变量偏移（Internal Covariate Shift）**问题。

### （5）位置编码
Transformer的核心是注意力机制。通过计算序列中任意两个词元的关系来捕捉依赖。它本身不关心任何关于词元顺序或位置的信息。为了解决这个问题，引入了**位置编码（Position Encoding）**。核心思想是：为输入序列的每个词元嵌入向量，都额外加上一个能够代表其绝对位置的和相对位置信息的“位置向量”。这个向量通过数学公式计算得到。

$ PE_{(pos，2i)}=sin(\frac{pos}{10000^{2i/d_{model}}}) $,

$ PE_{(pos，2i+1)}=cos(\frac{pos}{10000^{2i/d_{model}}}) $

其中：

+ pos是词元在序列中的位置
+ i是位置向量中的维度索引（从0到$ d_{model}/2 $）
+ $ d_{model} $是词嵌入向量的维度



## 3.1.3 Decode-only架构
Transformer的设计哲学是“先理解，再生成”。OpenAI在开发GPT时，提出更简单的思想：语言的核心任务是预测下一个更有可能出现的词。基于这个思想，GPT做了大胆的简化：抛弃了编码器，保留了部分解码器。这就是Decode-only架构的由来。

Decode-only架构的工作模式被称为**自回归（Autoregressive）**，过程如下：

+ 给模型一个起始文本（例如“Datawhale Agent is”）
+ 模型预测出下一个更有可能的词
+ 模型将自己刚刚生成的词添加到输入文本结尾，形成新的输入
+ 模型基于新的输入，再次预测下一个词
+ 不断重复这个过程，直到生成完整的句子或者达成停止条件

模型如何做到生成第t个词时，不去“偷看”第t+1个词的答案？

**掩码自注意力机制（Masked Self-Attention）**，工作原理：在自注意力机制计算出注意力分数矩阵（即每个词对其他词的关注度得分后），在进行softmax归一化之前，模型会应用一个“掩码”，这个掩码会将所有位于当前位置之后（即当前未观测到）的词元对应的分数，替换为一个非常大的负数。当这个带有负无穷分数的矩阵经过softmax函数时，这些位置的概率会变成0,。这样一来，模型在计算任何一个位置的输出时，都从数学上被阻止关注后面的信息。

优势：

+ 训练目标统一：模型的唯一任务就是“预测下一个词”，非常适合在海量的无标注文本数据上进行预训练
+ 结构简单，易于扩展：今天的GPT-4、LIama都是就这个架构
+ 天然适合生成任务：工作模式和所有的生成式任务（对话、协作、代码生成等）完美契合。

# 3.2 与大模型交互
## 3.2.1 提示词工程
**提示（Prompt）工程**就是研究如何设计出精准的提示，从而引导模型产生符合我们期望输出的回复。

### （1）模型采样参数
**Temperature**这类可配置的参数，本质是通过调整模型对“概率分布”的采样策略，让输出匹配场景需要。

传统的概率分布是softmax函数计算得到的。

+ Temperature：温度是控制模型输出“随机性”与“确定性”的关键参数。原理是引入温度系数$ T\gt0 $，将 Softmax 改写为$ p_i^{(T)} = \frac{e^{z_i / T}}{\sum_{j=1}^k e^{z_j / T}} $
    - 低温度（$ 0≤Temperature＜0.3 $）时，输出更“精准，确定”。适用场景如下：
        * 事实性任务：问答、数据计算、代码生成
        * 严谨性场景：法律条文解读、技术性文档撰写、学术性概念解读
    - 中温度（$ 0.3≤Temperature＜0.7 $）时，输出“平衡，自然”。适用场景如下：
        * 日常对话：客服交互、聊天机器人
        * 常规创作：邮件撰写、产品文案、简单故事创作
    - 低温度（$ 0.7≤Temperature＜2 $）时，输出“创新、发散”。适用场景如下：
        * 创意性任务：诗歌创作、科幻故事构思、广告Slogan、艺术灵感启发等
+ Top-K：原理是将所有的token按概率从高到低排序，取排名前K的token组成“候选集”，随后对筛选出的K个token的概率进行归一化$ \hat{p}i = \frac{p_i}{\sum{j \in \text{候选集}} p_j} $。
    - 与温度采样的区别和联系：温度采样通过温度T调整所有token的概率分布，不改变候选token的数量。top-k采样通过k值控制候选token的数量，再从中采样。当k=1时输出完全确定，退化为“贪心采样”。
+ Top-p：原理是将所有的token按概率从高到低排序，从排序后的第一个token开始，逐步累加概率，直到累加和首次达到或超过阈值p：$ \sum_{i=1}^{n} p_i≥P $。此时累加过程所有token组成“核集合”，最后对核集合归一化。
    - 与Top-K的区别与联系：相比于固定截断，对动态适应不同分布的“长尾”特性，对概率分布极不均衡的情况适应性更好。

### （2）零样本、单样本和少样本提示
根据给模型提供示例的数量，提示可以分为三个类型。

+ **零样本提示（Zero-shot Prompting）**：不给模型任何示例，直接让他根据指令完成任务。
+ **单样本提示（One-shot Prompting）**：给模型提供一个完整示例，向他展示任务的格式和期望的输出风格。
+ **少样本提示（Few-shot Prompting）**：提供多个示例，让模型更准确地理解任务的细节、边界和细微差别。

### （3）指令调优的影响
**指令调优（Instruction Tuning）**是一种微调技术，使用大量“指令-回答”格式的数据对预训练模型进行进一步的训练。经过指令调优后，模型能更好地理解用户的指令。我们今天日常工作学习中所有的模型（ChatGPT、DeepSeek、Qwen）都是经过指令调优的。

+ 对“文本补全”模型的提示（你需要用少样本提示“教会”模型做什么）

```plain
这是一段将英文翻译成中文的程序。
英文:Hello
中文:你好
英文:How are you?
中文:
```

+ 对“指令调优”模型的提示（你可以直接下达指令）：

```plain
请将下面的英文翻译成中文:
How are you?
```

### （4）基础提示技巧
**角色扮演（Role-playing）**通过赋予模型一个角色，可以引导它的回答风格、语气和知识范围，使其输出更符合特定场景的需求。

```plain
# 案例
你现在是一位资深的Python编程专家。请解释一下Python中的GIL（全局解释器锁）是什么，
要让一个初学者也能听懂。
```

**上下文示例（In-context Example）**这与少样本提示的思想一致，通过在提示中提供清晰的输入输出示例，来教会模型如何处理我们的请求，在处理复杂格式或特定风格的任务时非常有用。

```plain
# 案例
我需要你从产品评论中提取产品名称和用户情感。请严格按照下面的JSON格式输出。

评论:这款“星尘”笔记本电脑的屏幕显示效果惊人，但我不太喜欢它的键盘手感。
输出:{"product_name": "星尘笔记本电脑", "sentiment": "混合"}

评论:我刚买的“声动”耳机音质很棒，续航也超出了我的预期！
输出:
```

### （5）思维链
思维链（Chain-of-Thought，CoT）是一种强大的提示技巧，通过引导模型一步步思考，提升了模型在复杂任务的推理能力。

实现CoT的关键是在提示词中加一句简单的引导语，如“请逐步思考”或“Let's think step by step”

```plain
# 思维链提示
一个篮球队在一个赛季的80场比赛中赢了60%。在接下来的赛季中，他们打了15场比赛，赢了12场。
两个赛季的总胜率是多少？
请一步一步地思考并解答。

>>>
（模型可能会输出）
好的，我们来一步步计算。
第一步:计算第一个赛季赢得的比赛数。
80场 * 60% = 48场。
第二步:计算两个赛季的总比赛数和总胜利数。
总比赛数 = 80 + 15 = 95场。
总胜利数 = 48 + 12 = 60场。
第三步:计算总胜率。
总胜率 = (总胜利数 / 总比赛数) * 100% = (60 / 95) * 100% ≈ 63.16%。
所以，两个赛季的总胜率约为63.16%。
```

## 3.2.2 文本分词
 在将自然语言喂给大模型之前，必须将其转换成模型能够处理的数字格式。这个将文本转为数字序列的过程就是**分词（Tokenization）**。分词器（Tokenizer）的作用就是定义一套规则，将原始文本切分成一个个最小的单元，称之为词元（Token）

### （1）为什么要分词/分词策略的发展
+ **按词分词（Word-based）**：直接用空格或者标点符号将句子切分成单词。面临挑战：
    - 词表爆炸与未登录词：一个语言的词汇量巨大，每个词都作为独立词元，词表难以管理。模型无法处理任何未在词表中出现过的词，此现象叫做**“未登录词”（Out-Of-Vocabulary，OOV）**
    - 语义关联的缺失：模型难以捕捉词形相近的词之间的语义关系。尽管他们有相同的含义，例如“look”、“looks”与“looking”会被认为是三个不同的词元。同样训练数据中低频词由于出现次数少，其语义也难以被模型充分学习。
+ **按字符分词（Character-based）**：将文本分割成单个字符。这种方法词表很小（例如英文字母、数字、标点），不存在OOV问题。但缺点是：单个字符大多不具备独立的语义，模型需要花费更多的精力去学习如何将字符组成有意义的词。

为了兼顾词表大小和语义表达，现在大语言模型普遍使用**子词分词（Subword Tokenization）算法**。核心思想是：将常见的词保留为完整的词元，将不常见的词拆分成多个有意义的子词片段（如Agent是词元，Tokenization拆分为Token和ization）。既控制了词表大小，又能够让模型通过组词来理解和生成新词。

### （2）字节对编码算法解析
  **字节对编码（Byte-Pair Encoding，BPE）**是最主流的子词分词算法之一。核心思想：

+ 初始化：将词表初始化为所有在语料库中出现过的基本字符
+ 迭代合并：在语料库上，统计所有相邻词元对出现的频率，找到频率最高的一对，将它们合并为一个新的词元，加入到词表。
+ 重复：重复第二步，直到词表大小达到预设的阈值。

后续许多算法都是基于BPE基础上优化。

+ WordPiece：Google BERT模型采用的算法。合并词元的标准不是最高频率，而是“能最大化提升语料库语言模型概率”。简单来说，会优先合并哪些让整个语料库“通顺度”提升最大的词元对。
+ SentencePiece：LIama采用此算法。最大的特点，将空格也作为一个普通字符（一般用_表示）。使得分词和解码过程完全可逆，不依赖特定语言。

### （3）分词器对开发者的意义
+ 上下文窗口限制：模型上下文窗口以Token数量计算。同样一段话，在不同语言或者不同分词器下，Token数量差异巨大。精确管理输入长度，避免上下文限制是构建长时记忆智能体的基础。
+ API成本：大多数模型API按照Token数量计费。了解文本如何分词，有助于进一步控制成本。
+ 模型表现得异常：有时模型奇怪的表现根源在于分词。例如模型擅长计算2+2，但对于2 + 2（有空格）会计算出错。同样一个词因首字母大写，也可能被切分为完全不同的Token序列。在设计提示词和解析模型输出时，考虑这些“陷阱”有助于提升智能体鲁棒性。

## 3.2.3 调用开源大模型
Hugging Face Transformers是一个强大的开源库，提供了标准API来加载和使用数以万计的预训练模型。

## 3.2.4 模型的选择
### （1）模型选型的关键考量
+ 性能与能力：最核心的考量。不同的模型擅长的不同，有的是逻辑推理和代码生成，有的是创意写作。建议参考公开的基准测试的排行榜来评估模型的综合能力。
+ 成本：对于闭源模型，成本主要体现在API调用费用，通常按照Token计费。对于开源模型，成本则体现在本地部署所需的硬件（GPU和内存）和运维。
+ 速度（延迟）：一些轻量级或经过优化的模型在延迟上表现更优。
+ 上下文窗口：模型一次性能处理的Token上限。对于需要理解长文档、分析代码库或维持长期对话记忆的智能体，选择一个拥有较大上下文窗口的模型必要。
+ 部署方式：API调用最简单，但是数据需要发送给第三方。本地部署可以保证数据隐私和最高程度的自主可控。
+ 生态与工具链：主流模型拥有更丰富的社区、教程、预训练模型、微调工具和兼容的开发框架（比如LangChain）
+ 可微调性与定制化：对于需要处理特定领域的智能体来说，模型的微调能力至关重要。
+ 安全与伦理性：考虑大模型在偏见、毒性、幻觉方面的表现。

# <font style="color:rgb(15, 17, 21);">3.3 大模型语言的缩放法则与局限性</font>
### <font style="color:rgb(15, 17, 21);">3.3.1 缩放法则（Scaling Laws）</font>
+ **<font style="color:rgb(15, 17, 21);">基本规律</font>**<font style="color:rgb(15, 17, 21);">：模型性能（用损失衡量）与参数量、训练数据量、计算资源之间呈可预测的幂律关系。增加资源就能平稳提升性能。</font>
+ **<font style="color:rgb(15, 17, 21);">早期误区</font>**<font style="color:rgb(15, 17, 21);">：最初认为“模型越大越好”，但 DeepMind 的</font><font style="color:rgb(15, 17, 21);"> </font>**<font style="color:rgb(15, 17, 21);">Chinchilla 定律</font>**<font style="color:rgb(15, 17, 21);"> </font><font style="color:rgb(15, 17, 21);">修正了这一观点：在固定计算预算下，</font>**<font style="color:rgb(15, 17, 21);">模型应适当小一些，但要用更多数据训练</font>**<font style="color:rgb(15, 17, 21);">，才能达到最优性能。</font>
+ **<font style="color:rgb(15, 17, 21);">能力涌现</font>**<font style="color:rgb(15, 17, 21);">：当模型规模超过某个阈值后，会突然出现小模型不具备的能力，如链式思考、指令遵循、多步推理等。对智能体来说，足够大的模型是实现复杂自主行为的前提。</font>

---

### <font style="color:rgb(15, 17, 21);">3.3.2 模型的局限性与幻觉</font>
+ **<font style="color:rgb(15, 17, 21);">模型幻觉</font>**<font style="color:rgb(15, 17, 21);">：模型生成与事实、上下文矛盾的内容，本质是“编造”信息。常见类型：</font>
    - **<font style="color:rgb(15, 17, 21);">事实性幻觉</font>**<font style="color:rgb(15, 17, 21);">：与现实不符。</font>
    - **<font style="color:rgb(15, 17, 21);">忠实性幻觉</font>**<font style="color:rgb(15, 17, 21);">：未忠实反映输入（如摘要错误）。</font>
    - **<font style="color:rgb(15, 17, 21);">内在幻觉</font>**<font style="color:rgb(15, 17, 21);">：与输入直接矛盾。</font>
+ **<font style="color:rgb(15, 17, 21);">产生原因</font>**<font style="color:rgb(15, 17, 21);">：</font>
    - <font style="color:rgb(15, 17, 21);">训练数据包含错误或偏见。</font>
    - <font style="color:rgb(15, 17, 21);">自回归生成缺乏事实核查。</font>
    - <font style="color:rgb(15, 17, 21);">复杂推理时逻辑出错。</font>
+ **<font style="color:rgb(15, 17, 21);">其他局限</font>**<font style="color:rgb(15, 17, 21);">：知识时效性（无法知晓训练后的事件）、数据偏见（可能放大社会偏见）。</font>
+ **<font style="color:rgb(15, 17, 21);">缓解策略</font>**<font style="color:rgb(15, 17, 21);">：</font>
    - **<font style="color:rgb(15, 17, 21);">数据层面</font>**<font style="color:rgb(15, 17, 21);">：清洗数据、引入事实知识、使用 RLHF 等。</font>
    - **<font style="color:rgb(15, 17, 21);">模型层面</font>**<font style="color:rgb(15, 17, 21);">：改进架构、让模型表达不确定性。</font>
    - **<font style="color:rgb(15, 17, 21);">推理/生成层面</font>**<font style="color:rgb(15, 17, 21);">：</font>
        * **<font style="color:rgb(15, 17, 21);">检索增强生成（RAG）</font>**<font style="color:rgb(15, 17, 21);">：从外部知识库检索信息后再生成。</font>
        * **<font style="color:rgb(15, 17, 21);">多步推理与自我验证</font>**<font style="color:rgb(15, 17, 21);">。</font>
        * **<font style="color:rgb(15, 17, 21);">调用外部工具</font>**<font style="color:rgb(15, 17, 21);">（搜索引擎、计算器等）获取实时或精确信息。</font>

**<font style="color:rgb(15, 17, 21);">总结</font>**<font style="color:rgb(15, 17, 21);">：缩放法则告诉我们，资源投入可预测地提升性能，且要注重数据与参数的合理配比；而幻觉等局限性是当前 LLM 的固有挑战，需要通过数据、模型和推理策略多管齐下来缓解，从而构建更可靠的智能体。</font>

# <font style="color:rgb(31, 35, 40);">3.4 本章小结</font>
<font style="color:rgb(31, 35, 40);">本章介绍了构建智能体所需的基础知识，重点围绕作为其核心组件的大语言模型 (LLM) 展开。内容从语言模型的早期发展开始，详细讲解了 Transformer 架构，并介绍了与 LLM 进行交互的方法。最后，本章对当前主流的模型生态、发展规律及其固有局限性进行了梳理。</font>

**<font style="color:rgb(31, 35, 40);">核心知识点回顾：</font>**

+ **<font style="color:rgb(31, 35, 40);">模型演进与核心架构</font>**<font style="color:rgb(31, 35, 40);">：本章追溯了从统计语言模型 (N-gram) 到神经网络模型 (RNN, LSTM)，再到奠定现代 LLM 基础的 Transformer 架构。通过“自顶向下”的代码实现，本章拆解了 Transformer 的核心组件，并阐述了自注意力机制在并行计算和捕捉长距离依赖中的关键作用。</font>
+ **<font style="color:rgb(31, 35, 40);">与模型的交互方式</font>**<font style="color:rgb(31, 35, 40);">：本章介绍了与 LLM 交互的两个核心环节：提示工程 (Prompt Engineering) 和文本分词 (Tokenization)。前者用于指导模型的行为，后者是理解模型输入处理的基础。通过本地部署并运行开源模型的实践，将理论知识应用于实际操作。</font>
+ **<font style="color:rgb(31, 35, 40);">模型生态与选型</font>**<font style="color:rgb(31, 35, 40);">：本章系统地梳理了为智能体选择模型时需要权衡的关键因素，并概览了以 OpenAI GPT、Google Gemini 为代表的闭源模型和以 Llama、Mistral 为代表的开源模型的特点与定位。</font>
+ **<font style="color:rgb(31, 35, 40);">法则与局限</font>**<font style="color:rgb(31, 35, 40);">：本章探讨了驱动 LLM 能力提升的缩放法则，阐述了其背后的基本原理。同时，本章也分析了模型存在的如事实幻觉、知识过时等固有局限性，这对于构建可靠、鲁棒的智能体至关重要。</font>

**<font style="color:rgb(31, 35, 40);">从 LLM 基础到构建智能体：</font>**

<font style="color:rgb(31, 35, 40);">这一章的LLM基础主要是为了帮助大家更好的理解大模型的诞生以及发展过程，其中也蕴含了智能体设计的部分思考。例如，如何设计有效的提示词来引导 Agent 的规划与决策，如何根据任务需求选择合适的模型，以及如何在 Agent 的工作流中加入验证机制以规避模型的幻觉等问题，其解决方案均建立在本章的基础之上。我们现在已经准备好从理论转向实践。在下一章，我们将开始探索智能体经典范式构建，将本章所学的知识应用于实际的智能体设计之中。</font>
